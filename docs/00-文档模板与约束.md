# 模块文档模板与约束

所有模块文档统一遵循以下章节结构：

1. 模块目标
2. 职责边界
3. 模块结构
4. 核心流程
5. 接口约束
6. 数据约束
7. 缓存、事务与一致性约束
8. 安全约束
9. 异常、日志与审计约束
10. 实现边界
11. 开发顺序

编写要求：

- 使用 Markdown 标题结构，章节名称尽量保持一致。
- 只写设计方案、职责边界和实现约束，不写示例代码。
- 每个接口说明至少包含用途、请求路径、关键参数、幂等性和权限要求。
- 每个数据约束至少包含主实体、关键字段、状态流转或索引约束。
- 涉及跨模块协作时，明确“依赖谁”“禁止依赖谁”。
- 所有高风险链路必须写出失败处理、补偿策略或降级方案。
- 文档面向后续开发使用，约束必须可执行，避免空泛描述。
