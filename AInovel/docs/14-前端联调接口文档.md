# 前端联调接口文档

本文档面向前端联调，基于当前后端代码中已经实现的
Controller 生成。文档只保留联调阶段最需要的信息：请求路径、请求
方法、认证要求、关键参数、返回 `data` 结构，以及当前联调状态。

## 1. 文档范围

本文档覆盖当前项目中的 HTTP 接口，不包含 WebSocket 事件与底层
基础设施细节。当前项目未发现独立的 Router 或函数式路由定义，
以下接口均来自 Spring MVC Controller。

联调状态按以下口径标记：

- `可联调`：接口已暴露，前端可以直接调用。
- `占位实现`：接口可以调用，但当前返回示例数据或空成功结果。
- `当前未放行`：接口已定义，但按当前安全配置会被拒绝访问。

## 2. 公共约定

本节说明所有接口共享的调用规则。前端在开始联调前，先按这里的
约定封装请求层，可以减少重复适配成本。

### 2.1 基础路径

当前接口按用户侧与管理侧分组。管理侧同时保留两套前缀，二者语义
一致，前端任选其一即可。

| 类型 | 路径前缀 | 说明 |
|---|---|---|
| 用户侧 | `/api/v1` | App、Web、H5 主要使用 |
| 管理侧 | `/api/admin/v1` | 管理后台接口 |
| 管理侧兼容前缀 | `/api/v1/admin` | 与上面等价 |

### 2.2 认证与请求头

需要登录的接口使用 Bearer Token。部分写接口还支持或要求幂等键。

| 请求头 | 是否常用 | 说明 |
|---|---|---|
| `Authorization` | 是 | 格式为 `Bearer <accessToken>` |
| `X-Client` | 是 | 客户端类型：用户端用 `pc-web`（或 `h5`）；管理端用 `admin` |
| `Idempotency-Key` | 部分接口 | 写接口防重复提交 |
| `X-Device-Id` | 个别接口 | 设备标识 |

认证示例：

```http
Authorization: Bearer <accessToken>
X-Client: pc-web
```

### 2.3 统一响应结构

所有接口统一返回 `Result<T>`。前端只需要先判断 `code` 是否为 `0`，
再读取 `data`。

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

字段说明如下：

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | `int` | 业务状态码，`0` 表示成功 |
| `message` | `string` | 响应消息 |
| `data` | `object \| null` | 业务数据 |

### 2.4 分页结构

分页接口的 `data` 统一为 `PageResponse<T>`。前端可以复用统一分页模型。

```json
{
  "records": [],
  "total": 0,
  "page": 1,
  "size": 20
}
```

### 2.5 常用枚举

当前前端会直接接触到的枚举值不多，先列出联调阶段最常用的几组。

| 枚举 | 取值 |
|---|---|
| `TargetType` | `NOVEL`、`CHAPTER` |
| `MessageType` | `SYSTEM`、`INTERACTION`、`AUDIT` |
| `ReviewDecision` | `PASS`、`REJECT` |
| `NovelStatus` | `DRAFT`、`PENDING_AUDIT`、`REJECTED`、`PUBLISHED`、`ON_SHELF`、`OFF_SHELF`、`BANNED` |
| `AuditStatus` | `PENDING`、`PASS`、`REJECT`、`MANUAL_REVIEW`、`FAILED` |
| `AuditBizType` | `BIZ_NOVEL_INTRO`、`BIZ_CHAPTER_CONTENT`、`BIZ_COMMENT` |

### 2.6 通用错误码（后端 `StandardErrorCode`）

| `code` | `message` | 说明 |
|---|---|---|
| `1000` | `invalid request` | 通用请求非法（参数组合不合法、资源不存在等） |
| `1001` | `validation failed` | 参数校验失败（`@Valid` / `@Validated`） |
| `1002` | `invalid client header` | `X-Client` 缺失或与路径域不匹配 |
| `1003` | `idempotency key required` | 接口要求幂等键但未提供 |
| `2001` | `unauthenticated` | 未登录、Token 无效/过期、刷新令牌失效 |
| `2002` | `token expired` | 令牌过期（预留码） |
| `2003` | `forbidden` | 权限不足 |
| `2004` | `account disabled` | 账号禁用 |
| `2101` | `rate limited` | 限流拦截（预留码） |
| `2102` | `risk blocked` | 风控拦截（预留码） |
| `3000` | `business state invalid` | 业务状态不允许当前操作 |
| `3001` | `idempotent conflict` | 幂等冲突或重复提交 |
| `3002` | `content not visible` | 内容不可见（审核/上下架限制） |
| `3003` | `lock acquire failed` | 分布式锁获取失败 |
| `5000` | `internal server error` | 未处理异常 |
| `5001` | `dependency unavailable` | 依赖服务不可用（预留码） |
| `5002` | `task execution failed` | 任务执行失败（预留码） |

### 2.7 失败响应示例（Result<T>）

以下示例均符合统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

1. 鉴权失败（`GET /api/v1/users/me`）

```json
{
  "code": 2001,
  "message": "unauthenticated",
  "data": null
}
```

2. 参数校验失败（`POST /api/v1/auth/register`）

```json
{
  "code": 1001,
  "message": "username size must be between 4 and 32",
  "data": null
}
```

3. `X-Client` 非法（`GET /api/admin/v1/novels`）

```json
{
  "code": 1002,
  "message": "invalid client header",
  "data": null
}
```

4. 幂等冲突（`POST /api/v1/comments` 重复提交）

```json
{
  "code": 3001,
  "message": "idempotent conflict",
  "data": null
}
```

5. 业务状态非法（`POST /api/v1/novels/{novelId}/on-shelf`）

```json
{
  "code": 3000,
  "message": "business state invalid",
  "data": null
}
```

6. 内部错误（任意接口未捕获异常）

```json
{
  "code": 5000,
  "message": "internal server error",
  "data": null
}
```

## 3. 用户与认证模块

本模块负责注册、登录、令牌刷新、个人资料和站内信。前端通常先联调
这里，再进入阅读、互动和创作链路。

| 接口 | 方法 | 认证 | 功能 | 关键请求 | `data` 结构 | 联调状态 |
|---|---|---|---|---|---|---|
| `/api/v1/auth/register` | POST | 匿名 | 注册并返回令牌 | `username,password,nickname?,avatarUrl?,deviceId?` | `AccessTokenVO` | 可联调 |
| `/api/v1/auth/login` | POST | 匿名 | 登录并返回令牌 | `username,password,deviceId?` | `AccessTokenVO` | 可联调 |
| `/api/v1/auth/logout` | POST | 登录 | 退出登录 | 无 | `null` | 可联调 |
| `/api/v1/auth/refresh` | POST | 匿名 | 刷新访问令牌 | `refreshToken,deviceId` | `AccessTokenVO` | 可联调 |
| `/api/v1/users/me` | GET | 登录 | 获取当前用户信息 | 无 | `UserMeVO` | 可联调 |
| `/api/v1/users/me/profile` | PUT | 登录 | 更新个人资料 | `nickname?,avatarUrl?,bio?` | `UserProfileVO` | 可联调 |
| `/api/v1/users/me/messages` | GET | 登录 | 分页查询站内信 | `page,size,type,readStatus` | `PageResponse<UserMessageVO>` | 可联调 |
| `/api/v1/users/me/messages/read` | POST | 登录 | 批量标记已读 | `messageIds` | `null` | 可联调 |
| `/api/v1/internal/messages` | POST | `INTERNAL` | 内部消息投递 | `Idempotency-Key` 必填；`toUserId,type,title,content,bizType,bizId,producer,traceId` | `null` | 可联调 |

常用返回对象如下：

| 对象 | 关键字段 |
|---|---|
| `AccessTokenVO` | `accessToken,refreshToken,expireAt,loginVersion` |
| `UserMeVO` | `userId,username,status,roles,profile` |
| `UserProfileVO` | `nickname,avatarUrl,bio,level,verifiedStatus` |
| `UserMessageVO` | `messageId,type,title,content,bizType,bizId,read,createdAt` |

登录示例如下：

```http
POST /api/v1/auth/login
Content-Type: application/json
```

```json
{
  "username": "alice001",
  "password": "12345678",
  "deviceId": "web-6b0d6a53-0cbe-4f8f-83e9-d8b7dd1b27c1"
}
```

|---|---|---|---|---|---|---|

| `/api/v1/novels/{novelId}/chapters` | POST | 登录 | 新增章节 | `Idempotency-Key` 可选；`chapterNo,title,content` | `NovelChapterVO` | 可联调 |

| `/api/v1/novels/{novelId}/submit-audit` | POST | 登录 | 提交审核 | `Idempotency-Key` 可选；`chapterIds?,reason?` | `string` | 可联调 |

## 9. 当前联调限制

本节汇总当前代码里会直接影响前端联调的已知差异。前端在接入前先看
这里，可以避免把“接口未生效”误判为请求代码问题。

- 推荐相关接口当前返回示例数据，不代表真实推荐结果。
- 点赞、收藏、关注接口已切换为真实落库实现；管理端隐藏评论仍为占位。
- 管理端小说列表接口已切换为真实查询，支持筛选、关键词和分页。
- `/api/v1/counters` 已放行，支持匿名计数查询和登录态状态回显。
- `/api/v1/{targetType}/{targetId}/comments` 已定义，但当前安全配置会拒绝
  访问。
- `/api/v1/audit/text` 已定义，但当前安全配置会拒绝访问。
- `/api/v1/audit/tasks/{taskId}` 已定义，但当前安全配置会拒绝访问。
