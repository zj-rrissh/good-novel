# 小说爬虫导入说明

如果你准备在仓库外部单独开发爬虫程序，并直接写数据库，请优先阅读：

- [16-外部爬虫数据入库对接文档.md](/mnt/d/MyProgram/xiaoshuo/AI-novel/AInovel/docs/16-外部爬虫数据入库对接文档.md)

## 目标

当本地数据库没有小说资源时，可以通过后端内置的“可配置爬虫导入器”抓取目标站点的小说详情页、目录页和章节正文，并按现有表结构写入：

- `novel`
- `novel_chapter`

导入时会自动维护：

- `novel.status`
- `novel.latest_chapter_id`
- `novel.word_count`
- `novel_chapter.status`
- `novel_chapter.published_at`

## 使用前提

- 先确认目标站点允许抓取，且你对抓取内容有使用权限。
- 本地 MySQL / Redis 已启动，后端可正常连库。
- 默认作者 ID 为 `1002`（`author_demo`），也可以在配置文件里给每本书单独指定 `authorId`。

## 配置文件

示例文件见：

- [novel-spider-config.sample.json](/mnt/d/MyProgram/xiaoshuo/AI-novel/AInovel/docs/examples/novel-spider-config.sample.json)

核心字段说明：

- `detailUrl`：小说详情页 URL
- `titleSelector`：小说标题 CSS 选择器
- `introSelector`：小说简介 CSS 选择器，可选
- `coverSelector` / `coverAttr`：封面选择器和属性名，可选
- `chapterLinkSelector` / `chapterLinkAttr`：目录页章节链接选择器和属性名
- `chapterOrder`：目录顺序，`ASC` 或 `DESC`
- `chapterTitleSelector`：章节页标题选择器，可选；为空时回退到目录链接文案
- `chapterContentSelector`：章节正文选择器
- `categoryId` / `tagIds`：直接映射到数据库字段
- `novelStatus`：默认 `ON_SHELF`
- `maxChapters`：限制导入章节数，可选

## 运行方式

在 `AI-novel/AInovel` 目录执行：

```bash
./mvnw spring-boot:run \
  -Dspring-boot.run.arguments="--ainovel.spider.enabled=true --ainovel.spider.spec-file=/absolute/path/to/novel-spider-config.json"
```

可选参数：

```bash
--ainovel.spider.default-author-id=1002
--ainovel.spider.request-timeout=15s
--ainovel.spider.user-agent=AInovelNovelSpider/1.0
--ainovel.spider.continue-on-error=true
```

## 导入行为

- 同一作者下，按 `title` 查重。
- 如果小说不存在，则新建 `novel` 和 `novel_chapter`。
- 如果小说已存在，则更新小说基础信息，并按 `chapter_no` 更新或新增章节，不重复插入。
- 导入章节会直接标记为 `PUBLISHED`，小说状态按配置写入，默认 `ON_SHELF`。

## 限制

- 当前版本适合“一个详情页 + 一个目录选择器 + 一个章节正文选择器”的常规站点。
- 暂不支持登录态抓取、JS 动态渲染页面、章节分页合并、反爬验证码处理。
